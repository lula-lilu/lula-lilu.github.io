如何做一份优秀的用户体验报告

Posted on 2017-05-03

过了好几天才回过头来总结，来源是起点学院的公开课，做分享的是百度糯米的PM。

听课的过程加上后面跟着PPT回顾了一下，做了下面这个思维导图，整体讲的内容框架还是挺全的。第一部分讲了做一份用户体验报告的四个步骤，之后以猫眼电影和百度糯米的支付举了两个例子，更深入的分析那些地方是误区。下面是自己感触比较深的四个点。

  1. 因为自己目前针对的受众是面试官，最重要的是展现自己的能力，从一个点去突破才能让人印象深刻，而这个其实也是最难的部分。
  2. 纵向深挖的几个图：业务流程图，产品结构图，产品迭代图，任务分析法（感觉与用户体验地图有些类似，简化版）
  3. 从产品的视角，用户的视角而不是你自己的个人视角
  4. 有趣的标题+有力的结论

最开始写用户体验报告，以为自己经常用软件时都会有一些自己的小想法的应该也不难写，然后在知乎各种网站上看别人的报告，看完用户体验要素的各种框架就感觉无从下手，行业分析，市场分析对于我来说实在不知道怎么写。然后一次偶然看到了人人都是产品经理上很久之前的一篇文章，把产品分析转换为回答问题。

问题 框架
这个产品是干什么的？ 产品定位
这个产品是给谁用的？ 目标用户分析
这个产品解决了什么需求和痛点？ 需求分析
用什么方式解决的？ 产品结构图+任务流程图
这个产品有哪些做的好的地方？哪些做的不好的地方？ 产品分析【核心功能分析(针对的需求+功能分析+优化方案)】【辅助功能分析(针对的需求+功能分析+优化方案)】产品功能体验【界面】
如果换做你来做，你会这么做？ 我是产品经理总结
